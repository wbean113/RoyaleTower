## 一、开场白 & 项目概览（约 40秒）

**📺 展示文件：`gamescene.h` 第21-192行**

各位好，我今天要讲解的项目是 **RoyaleTower（皇家塔防）**——一个基于 **Qt 6.11** 的 2D 图形化塔防游戏。

先看整体架构。核心类是 `GameScene`，它继承自 `QGraphicsScene`，管理所有游戏对象。场景尺寸是 1600×900 像素，上方 720 像素是游戏地图区域，下方 180 像素是 UI 控制栏。地图采用 **20列 × 9行** 的网格系统，每个网格 80×80 像素。

大家看第 56-63 行，这些 `constexpr` 常量定义了整个游戏的空间坐标系。第 68 行的 `gridToScene` 是网格坐标到场景像素的转换函数，后面会详细讲。

**📺 切换到：`towerr.h` 第12-22行**

游戏中，`Tower` 类采用**多重继承**——同时继承 `QObject` 和 `QGraphicsRectItem`。`QObject` 提供信号槽能力用于右键出售，`QGraphicsRectItem` 提供场景中的图形渲染。我们定义了四种塔类型：箭塔快速低伤、加农炮慢速高伤、冰塔减速、魔法塔范围伤害。

**📺 切换到：`carditem.h` 第9-19行**

同样，`CardItem` 也采用 `QObject + QGraphicsRectItem` 的多重继承，代表底部的七张卡牌：四种塔加三种法术（火球术、冰冻术、闪电术）。

## 二、游戏主循环（约 50秒）

**📺 展示文件：`gamescene.cpp` 第529-624行（`updateGame` 函数）**

游戏的驱动力是定时器。在构造函数中，我们启动了四个核心定时器：`resourceTimer` 每 3 秒产出圣水、`spawnTimer` 每 1.5 秒出兵、`gameLoopTimer` 每 50 毫秒调用 `updateGame()`、`cooldownTimer` 每 200 毫秒更新卡牌冷却。

现在看 `updateGame` 这个核心函数。它每 50ms 执行一次，分四步走：

**第一步**（第533-576行）：遍历所有存活敌人。调用 `regenerate` 给 king2 回血、`updateStatusEffects` 更新冰冻/减速状态、`moveTowards` 向路径点移动。如果敌人到达终点且是 King，立即失败；普通敌人漏掉则扣血，漏掉 15 个游戏结束。

**第二步**（第579-581行）：让所有防御塔攻击范围内的敌人。

**第三步**（第584-615行）：清理已死亡敌人，发放击杀奖励。关键是第587-589行——如果被杀的是 King，触发胜利条件。

**第四步**（第618-623行）：更新 HUD 显示——波次、击杀数、在场敌人数。

## 三、核心算法详解（约 70秒）

### 3.1 网格坐标转换

**📺 展示文件：`gamescene.cpp` 第68行（头文件声明）、`gamescene.h` 第68行**

```cpp
static QPointF gridToScene(int col, int row) {
    return QPointF(col * GRID_SIZE + GRID_SIZE / 2.0,
                   row * GRID_SIZE + GRID_SIZE / 2.0);
}
```

这个函数将网格逻辑坐标 (col, row) 映射为场景像素中心点。比如第 0 列第 4 行 → 像素坐标 (40, 360)，当玩家点击地图放置塔时，`snapToNearestCell` 会找到最近的合法网格点吸附。

### 3.2 防御塔索敌算法

**📺 展示文件：`towerr.cpp` 第219-234行（`findTarget` 函数）**

```cpp
Enemy *Tower::findTarget(QList<Enemy *> &enemies) const
{
    Enemy *bestTarget = nullptr;
    qreal bestDist = range;
    for (Enemy *e : enemies) {
        if (e->isDead()) continue;
        qreal dist = QLineF(pos(), e->pos()).length();
        if (dist < bestDist) {
            bestDist = dist;
            bestTarget = e;
        }
    }
    return bestTarget;
}
```

遍历所有存活敌人，用 `QLineF::length()` 计算塔心到敌人的欧氏距离。一旦找到射程内距离最短的敌人，立即锁定为攻击目标。这是贪心策略——优先攻击最近的敌人。

### 3.3 敌人路径跟随

**📺 展示文件：`enemy.cpp` 第209-223行（`moveTowards` 函数）**

敌人通过 `waypoints` 路点列表沿预设路径移动。每帧调用 `moveTowards(target)`：先计算方向向量，用曼哈顿距离判断是否接近目标点。距离小于 `speed` 时直接吸附到路点；否则沿方向向量逐步移动。到达路点后调用 `advanceToNextWaypoint` 切换到下一个路点。这里不使用欧氏距离而用曼哈顿距离，是为了保证水平/垂直移动的步长稳定性。

### 3.4 魔法塔AOE伤害

**📺 展示文件：`towerr.cpp` 第191-205行**

魔法塔的特殊之处在于范围伤害。半径 250 像素内的所有敌人都会受到伤害，伤害随距离线性衰减：

```cpp
qreal rate = 1.0 - (dist / aoeRadius) * (1.0 - minDamageRate);
```

距离越近伤害越高，边缘最低保留 30% 伤害。这是一个简洁而有效的AOE伤害模型。

## 四、法术系统 & 波次系统（约 50秒）

**📺 展示文件：`gamescene.cpp` 第946-1011行**

三种法术各有特色：

- **火球术**（第946行）：点击位置为中心，半径 150，对所有范围内敌人造成 100 点伤害。曼哈顿距离判断范围。
- **冰冻术**（第964行）：全屏生效，半径 600，冻结所有敌人 4 秒。冻结期间敌人速度降为 0 且不能移动。
- **闪电术**（第981行）：聪明的设计——按敌人距离终点远近排序，优先打击**最靠后的 3 个敌人**，每人 70 伤害。这对快要到达终点的敌人特别有效。

**📺 向下滚动到第1028-1079行（波次系统）**

波次系统用 `randomEnemyType` 决定出兵类型。随着波次递增，大怪比例逐步提高。前两波主要是 Scout（小狗），第四波开始出现 Tank，第五波后加 Boss。关卡 2 和 3 在最终波次后还会生成 **King**——击败 King 才算通关。

**📺 滚动到第293-306行（`enemy.cpp` 中 `regenerate` 函数）**

值得一提的是 **king2 回血机制**：每 500ms 恢复 15 点血量，而且 king2 只承受 **10% 的伤害**（第228行 `damage = (damage + 9) / 10`），非常难被击杀。

## 五、内存安全——三重安全防护（约 30秒）

**📺 展示文件：`towerr.cpp` 第50-68行（析构函数）**

这是我在开发中踩过坑后总结的经验。防御塔攻击时会创建弹道轨迹和专属定时器，如果塔被出售或关卡切换时没清理干净，定时器回调会访问已销毁的内存，导致 **悬空指针崩溃**。

解决方案是三重安全防护，体现在析构函数中：

- **第1步**（第54-58行）：`stop + delete attackTimer`，立即停止主攻击定时器
- **第2步**（第59行）：`killAllTrailTimers()`，逐一停止并删除所有轨迹专属定时器
- **第3步**（第62行）：`cleanupTrails()`，同步删除所有残留的轨迹图形项

执行顺序严格保证：**先杀定时器，再清图形**。这样确保 100% 消除双重释放和野指针。

## 六、信号槽解耦 & 地图瓦片系统（约 25秒）

**📺 展示文件：`gamescene.cpp` 第671-726行与第1274-1317行**

塔的右键出售功能通过信号槽实现——`Tower::sellRequested` 信号连接到 `GameScene::onSellTower` 槽，返还 50% 圣水并安全清理塔的所有资源。

地图方面，我们有三套 20×9 的瓦片模板。路径瓦片编码为 1-8（垂直、水平、四种转弯、十字路口），装饰瓦片用 `-1` 占位，每次 `setupLevel` 时随机替换为 9-13 的五种变体，让地图每次开局都有不同的视觉效果。
